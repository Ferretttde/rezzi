## Architecture

### Routing
TanStack Router with file-based routing — routes live in `src/routes/`. The `src/routeTree.gen.ts` file is auto-generated by the Vite plugin on `npm run dev`; never edit it manually.

Route files export a `Route` constant via `createFileRoute(path)({ component })`. The root layout (`src/routes/__root.tsx`) handles the auth guard in `beforeLoad`, redirecting unauthenticated users to `/auth/login`.

### Data Fetching
All server state uses TanStack Query v5 (`src/hooks/use*.ts`). Each domain has a query key factory (e.g. `recipeKeys`, `plannerKeys`) and co-located fetch functions. Mutations call `queryClient.invalidateQueries` in `onSuccess`. Optimistic updates use `onMutate`/`onError`/`onSettled` (see `useToggleFavorite` in `src/hooks/useRecipes.ts`).

The QueryClient (`src/lib/queryClient.ts`) is configured with `gcTime: Infinity` in production for offline support.

### Supabase Integration
`src/lib/supabase.ts` exports a singleton client. All DB access goes through it — never instantiate another client. Ingredients and steps are stored as `JSONB` and must be cast via `as unknown as Json` when writing and cast back when reading.

Real-time sync (`src/hooks/useRealtimeSync.ts`) subscribes to `postgres_changes` on `recipes`, `meal_plans`, and `recipe_tags` tables, triggering `queryClient.invalidateQueries` on any change. It's mounted once in the root layout.

### UI State
Zustand (`src/store/uiStore.ts`) manages only ephemeral UI state: recipe list search query and selected tag IDs.

### Component Conventions
- UI primitives are Radix UI components wrapped in `src/components/ui/` (shadcn-style)
- Use `cn()` from `src/lib/utils.ts` (clsx + tailwind-merge) for conditional class merging
- Layout: `AppShell` wraps every authenticated page with offline banner + `BottomNav`
- The app is in German (UI strings are German)

### Data Model
- All tables have `household_id` with RLS enforced via `my_household_id()` security-definer function
- `recipes.ingredients` and `recipes.steps` are JSONB arrays typed as `Ingredient[]` and `RecipeStep[]` in `src/types/app.ts`
- Tags are many-to-many via `recipe_tags` join table
- `meal_plans` references `recipes` optionally (custom_title for ad-hoc entries)

### Photo Import (Claude Vision)
`/recipes/new` FAB → "Foto importieren" → `PhotoImportForm` → `usePhotoImport` → `import-recipe-image` Edge Function.
- Edge function calls Anthropic API directly via `fetch` (no SDK) using `ANTHROPIC_API_KEY` Supabase secret
- Must be deployed with `--no-verify-jwt` (Supabase gateway otherwise blocks POST before function runs)
- Image resized to max 1200px JPEG 0.85 quality before base64 encoding
- Returns same `ImportedRecipe` shape as URL import → pre-fills `RecipeForm`
